搭博客遇到的问题

标签（空格分隔）： hexo

今天搭建hexo博客，在最后一切准备就绪，部署到github上却找不到页面，最终搜索处理方法，得到了解决方法。主要注意以下几点：首先注意仓库名和github的用户名一致；其次注意配置vim_config.yml文件时

deploy:
type: git
repo: https://github.com/"yourgithubname"/"yourgithubname".github.io.git
branch: master

type后面跟git而不是github，另外注意每个冒号后面有半角的空格。

学习笔记12-26

标签（空格分隔）：java

1.String与Strinbuffer
当需要对字符串进行操作时，使用StringBuffer而不是String，String是read-only的，如果对它进行修改，会产生临时对象，而StringBuffer是可修改的，不会产生临时对象。

2.HashMap与HashTable
HashTable中的方法是同步的，HashTable的方法在缺省情况下是非同步的，因此在多线程环境下需要做额外的同步机制。
HashTable不允许有null值key和value，而HashTable允许有null值key和value，因此HashMap使用containKey（）来判断是否存在某个键。
HashTable是Dictionary的子类，HashMap是Map接口的一个实现类。

学习笔记12-25

标签（空格分隔）： web后台

1.数据库索引的结构
索引的实现通常使用B树及其变种B+树。
数据库系统还维护着满足特定查找算法的数据结构，这些数据结构以某种方式引用数据，这样就可以在这些数据结构上实现高级查找算法。
创建索引付出的代价：一是增加了数据库的存储空间，二是在插入和修改数据时要花费较多的时间（因为索引也要随之变动）
唯一索引：唯一索引是不允许其中任何两行具有相同索引值的索引。
主键索引：数据库表经常有一列或列组合，其值唯一标识表中的每一行，该列称为表的主键。
聚集索引：在聚集索引中，表中行的物理顺序与键值的逻辑（索引）顺序相同。一个表只能包含一个聚集索引。

2.指针和引用
相同点：都是地址的概念：指针指向一块内存，它的内容是所指内存的地址；引用是某块内存的别名。
区别：
1.指针是一个实体，而引用仅是个别名；
2.引用只能在定义时被初始化一次，之后不可变，指针可变；
3.引用没有const，指针有const，const的指针不可变；
4.引用不能为空，指针可以为空。

scp命令拷贝虚拟机中内容导致软链失效

标签（空格分隔）： Linux

在此处软链接又叫符号链接，这个文件包含了另一个文件的路径名。可以是任意文件或目录，可以链接不同文件系统的文件，用ln -s 命令可以生成一个软连接。
从其他虚拟机拷贝文件夹一般常用的是scp命令，例如：scp -r george@ccq:/test1/ftest2/* ./test3
是将机器ccp上/test1/ftest2/目录下的所有文件（包括目录）拷贝到本地当前目录下的test3目录中，但是需要注意的一点是，如果拷贝的文件夹中含有软链，那么用这个命令就会使得拷贝得来的软链无法使用，因为在用此命令拷贝的过程中，无法识别出软链，而是直接读取之后进行拷贝。
解决此问题可以换用另外一种命令进行拷贝，即rsync命令，加上-l参数可以实现只拷贝软连接。
